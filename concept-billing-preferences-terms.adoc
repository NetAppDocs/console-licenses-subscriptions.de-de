= Begriffe und Definitionen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Die Abrechnungseinstellungen steuern, wie die NetApp Console Zahlungen, Rechnungen und das Verlängerungsverhalten für Ihre Lizenzen und Abonnements handhabt.  Sie können Ihre Präferenzen auf einer von zwei Ebenen festlegen:

* *Global* — gilt für Direktlizenzen, Marktplatz-Abonnements und Keystone Abonnements.
* *Pro Abonnement* — gilt nur für einen bestimmten Marktplatz oder einen direkten Lizenzvertrag.


Die Rechnungsquelle kann folgende sein:

* *Marktplatz-Abonnements* — Die Abrechnung erfolgt über AWS Marketplace, Azure Marketplace oder Google Cloud Marketplace.  Sie können das Abrechnungskonto, die Region und die Verlängerungsoptionen auswählen, die vom Cloud-Marktplatz angeboten werden.
* *Direktlizenzen* — Die Abrechnung erfolgt über NetApp.  Sie verwalten Lizenzschlüssel, Rechnungskontakte und die automatische Verlängerung in der NetApp Console.
* * Keystone Abonnements* — Die Abrechnung erfolgt über Ihren Keystone Vertrag.  Sie verwalten Rechnungsempfänger und Kostenstellenkennzeichnungen in der NetApp Console.




== Berechtigungen

Sie benötigen die Rolle *Account Admin* oder *Billing Admin*, um Abrechnungseinstellungen zu ändern. Operatoren können die Einstellungen einsehen, aber nicht bearbeiten.



== Abrechnungseinstellungen festlegen

Mit den Abrechnungseinstellungen können Sie: - eine einheitliche Rechnungskontaktliste für alle Dienste erzwingen, - die Nutzung Kostenstellen zuordnen, - automatische Verlängerungs- und Ablaufbenachrichtigungen verwalten, - sicherstellen, dass neue Cloud Volumes ONTAP (CVO)-Systeme das richtige Abrechnungskonto übernehmen.



= Abrechnungsfelder



== Marktplatzfelder

* *Marktplatzanbieter* — AWS, Azure oder Google Cloud.
* *Abrechnungskonto / Abonnement-ID* — Das Cloud-Abrechnungskonto, das belastet wird.
* *Vertragslaufzeit* — Vertragsdauer und Verlängerungsverhalten.




= Abrechnungsoptionen und Definitionen



== Bedingungen und Optionen

* **NetApp licenses first** — Abrechnung auf Basis aktiver NetApp-Lizenzen. Zusätzliche Nutzung erfolgt über den Marktplatz.
* **Nur Marketplace-Abonnements** — Die gesamte Nutzung wird über Marketplace-Verträge abgerechnet.
* **PAYGO (Pay-As-You-Go)** — Flexibles Abonnement, abgerechnet nach tatsächlicher Nutzung.
* **Jahresvertrag** — Marktplatz-Abonnement mit fester Laufzeit.
* **Automatische Verlängerung** — Verlängert den Vertrag bei Ablauf automatisch.
* **Benutzerdefinierte CVO-Konfiguration** — Erweiterter Modus für die Zuordnung mehrerer Abonnements pro Cloud Provider.
* **Direktlizenz** — NetApp-verwaltete Lizenz, die vor der Marketplace-Kapazität verwendet wird.
* **Hyperscaler** — AWS-, Azure- oder Google Cloud-Anbieter.
* **Dialog zur Bestätigung der Änderungen** — Erscheint vor der Anwendung von Updates, um versehentliche Abrechnungsänderungen zu verhindern.




== Typische Konfigurationspfade
